# PCG liány: po povrchu i zavěšené

Dva pokročilé nástroje nad [PCG Mode](pcg-zaklady.md), oba od Procedural Minds a oba postavené na stejném vzoru „štětec = graf s tool tagem": liány, které **lezou po povrchu** libovolného meshe (mesh sampling + pathfinding), a liány **zavěšené mezi dvěma kliky** s automaticky spočteným průvěsem. Druhý recept je obecnější, než zní — je to nástroj na závěsné křivky: lanové mosty, vlajky, vánoční světla.

---

## Liány po povrchu: mesh sampler + pathfinding

**Zdroj:** [Spawn Vines On Any Mesh With A Single Click Using PCG Mode in UE5.7](https://www.youtube.com/watch?v=rYLF1up8sc8) · [Procedural Minds](https://www.youtube.com/channel/UCkfvcZJ6dGNUSCWOCVQnkXw) · ~26 min, tutoriál

**Shrnutí:** Klik na zeď — a liána se rozleze po povrchu, kopíruje ho a cestou vyroste v menší šlahouny s listím a květy [(0:01)](https://www.youtube.com/watch?v=rYLF1up8sc8&t=1s). Jádro: **Mesh Sampler** rozprostře body po meshích, **Pathfinding** uzel mezi nimi najde cestu z místa kliku k náhodnému cíli — a fitness score s density noisem zařídí, že cesta bloudí jako rostlina, ne jako GPS [(12:26)](https://www.youtube.com/watch?v=rYLF1up8sc8&t=746s).

### Rozpad myšlenky

**Vstupy:** vedle PCG je potřeba plugin **Geometry Script Interop for PCG** — tam bydlí Mesh Sampler [(0:01)](https://www.youtube.com/watch?v=rYLF1up8sc8&t=1s). Graf přijde do světa jako volume (liány vznikají uvnitř) a meshe, po kterých se smí lézt, dostanou actor tag `vine` [(0:47)](https://www.youtube.com/watch?v=rYLF1up8sc8&t=47s). Registrace štětce přes Tool Data (paint tool, preset) jako u [vlastního štětce se scatteringem](pcg-zaklady.md#opravdovy-stetec-scattering-vlastnim-grafem) [(1:34)](https://www.youtube.com/watch?v=rYLF1up8sc8&t=94s).

**Sampling meshů (loop subgraf):** meshe do grafu dostane **Get Actor Property** — all world actors by tag, select multiple, a property name s tečkovou cestou **`StaticMeshComponent.StaticMesh`**, protože proměnná nežije na aktorovi, ale na komponentě [(4:39)](https://www.youtube.com/watch?v=rYLF1up8sc8&t=279s) [(5:25)](https://www.youtube.com/watch?v=rYLF1up8sc8&t=325s); pozice aktorů dodá **Get Actor Data** v režimu single point [(6:11)](https://www.youtube.com/watch?v=rYLF1up8sc8&t=371s). V loopu pak **Mesh Sampler** s metodou **Poisson sampling** (sampling radius: větší číslo = řidší body) — a pozor, vzorkuje **v originu**, takže hned za ním patří **Copy Points** na skutečné pozice aktorů [(3:52)](https://www.youtube.com/watch?v=rYLF1up8sc8&t=232s). Gotcha: scale aktoru škáluje i rozestupy vzorků [(6:58)](https://www.youtube.com/watch?v=rYLF1up8sc8&t=418s).

**Od kliku k lianě:** kliky štětce čte Get Tool Data (instance `vine`) a **Attribute Partition podle `$index`** je rozdělí po jednom [(7:44)](https://www.youtube.com/watch?v=rYLF1up8sc8&t=464s) [(8:30)](https://www.youtube.com/watch?v=rYLF1up8sc8&t=510s). Loop „main vine" pro každý klik: **Distance (spatial)** od navzorkovaných bodů ke kliku → **filter range 100–400** (jak daleko liána doleze) → filter na **position.Z greater** než klik, ať liána stoupá → **Random Choice** (fixed, 1) vybere cíl [(9:17)](https://www.youtube.com/watch?v=rYLF1up8sc8&t=557s) [(10:50)](https://www.youtube.com/watch?v=rYLF1up8sc8&t=650s) [(11:38)](https://www.youtube.com/watch?v=rYLF1up8sc8&t=698s). Pak **Pathfinding**: start = klik, goal = vybraný bod, search distance ~30 (musí odpovídat hustotě samplingu — moc málo a cesta se nenajde) [(11:38)](https://www.youtube.com/watch?v=rYLF1up8sc8&t=698s) [(12:26)](https://www.youtube.com/watch?v=rYLF1up8sc8&t=746s). Přímou trasu rozbije **cost function mode = fitness score** + **Density Noise** (0–1) jako cost atribut — liána najednou bloudí [(12:26)](https://www.youtube.com/watch?v=rYLF1up8sc8&t=746s) [(13:12)](https://www.youtube.com/watch?v=rYLF1up8sc8&t=792s).

**Geometrie:** na pathfindingu vypnout *output spline* — body je potřeba **napřed zmenšit** (Transform Points 0,5–0,7; škálovat hotovou splinu nefungovalo) a teprve pak **Create Spline → Spline Mesh Spawner** (válec, vlastní materiál, **forward axis Z** — válec míří vzhůru) [(13:12)](https://www.youtube.com/watch?v=rYLF1up8sc8&t=792s) [(14:44)](https://www.youtube.com/watch?v=rYLF1up8sc8&t=884s).

**Sekundární šlahouny:** vzorkované body do 50 jednotek od hlavní liány (Distance + filter) → Select Points 20 % → pro každý najít **přípojku na lianě**: Distance s **output distance vector** a Add Attribute pozice + vektor = bod na lianě [(15:30)](https://www.youtube.com/watch?v=rYLF1up8sc8&t=930s) [(17:50)](https://www.youtube.com/watch?v=rYLF1up8sc8&t=1070s). Párování konců: `$index` zkopírovaný do obyčejného atributu (Copy Attributes), merge a **Attribute Partition podle něj** — každá dvojice zvlášť [(18:37)](https://www.youtube.com/watch?v=rYLF1up8sc8&t=1117s) [(19:23)](https://www.youtube.com/watch?v=rYLF1up8sc8&t=1163s). Rovné spojnice jsou mrtvé — proto **druhý, hustší sampling** (count 5 místo 10) a mini-loop jen s pathfindingem (Get Attribute From Point Index: 0 = start, 1 = cíl; search 20; fitness zpět na distance) [(20:10)](https://www.youtube.com/watch?v=rYLF1up8sc8&t=1210s) [(20:56)](https://www.youtube.com/watch?v=rYLF1up8sc8&t=1256s). Na šlahouny pak Static Mesh Spawner s listím a květy (scale 1–2) [(22:29)](https://www.youtube.com/watch?v=rYLF1up8sc8&t=1349s).

**Dvě pasti na závěr:** Megascans kusy v 5.7 zblízka přepínají do rozbitého LOD — pomůže **minimum LOD 0→1** [(23:15)](https://www.youtube.com/watch?v=rYLF1up8sc8&t=1395s). A po smazání všech kliků se celá scéna pokryla listím: Distance uzel bez targetu neculluje — **Data Count → Branch** (0 = stop) pustí graf dál jen, když je co kreslit [(24:01)](https://www.youtube.com/watch?v=rYLF1up8sc8&t=1441s) [(24:47)](https://www.youtube.com/watch?v=rYLF1up8sc8&t=1487s).

> **Pozn.:** Drobné zrychlovače z videa: podržet `M` + klik = Static Mesh Spawner, podržet `1` + klik = float konstanta. A pathfinding občas proleze skrz mesh na odvrácenou stranu — řešení je prozaické: smazat klik a kliknout znovu [(13:58)](https://www.youtube.com/watch?v=rYLF1up8sc8&t=838s).

## Visící liány: řetěz mezi dvěma kliky

**Zdroj:** [Spawn Hanging PCG Vines And More Using This Simple Setup](https://www.youtube.com/watch?v=5bbKHksth9Q) · [Procedural Minds](https://www.youtube.com/channel/UCkfvcZJ6dGNUSCWOCVQnkXw) · ~25 min, tutoriál

**Shrnutí:** Klik A, klik B — a mezi body se natáhne liána s průvěsem, jehož hloubku řídí min/max hang parametry a jehož **nejnižší bod se posouvá k nižšímu z obou konců**, jak by to udělala skutečná šňůra [(0:01)](https://www.youtube.com/watch?v=5bbKHksth9Q&t=1s) [(24:03)](https://www.youtube.com/watch?v=5bbKHksth9Q&t=1443s). Pod kapotou: párování kliků dělením indexu, lerp se spočtenou alphou a spline se třemi body.

### Rozpad myšlenky

**Párování kliků:** kliky přicházejí jako jedna řada bodů — **Divide** `$index / 2` do atributu `vine` dá 0; 0,5; 1; 1,5…, a místo floor stačí **right-click na výstupní pin → convert to Integer32**: 0, 0, 1, 1 [(3:08)](https://www.youtube.com/watch?v=5bbKHksth9Q&t=188s) [(3:55)](https://www.youtube.com/watch?v=5bbKHksth9Q&t=235s). Attribute Partition podle `vine` (explicitně, ne *last*) rozdělí body na dvojice [(4:42)](https://www.youtube.com/watch?v=5bbKHksth9Q&t=282s).

**Kde je dno průvěsu (loop subgraf):** dvojici rozdělí filter na `$index == 0` (první vs. druhý bod) [(7:03)](https://www.youtube.com/watch?v=5bbKHksth9Q&t=423s). Pak počty — autor otevřeně říká, že vzorec mu dala AI, a předvádí, že funguje [(5:30)](https://www.youtube.com/watch?v=5bbKHksth9Q&t=330s) [(10:07)](https://www.youtube.com/watch?v=5bbKHksth9Q&t=607s): výškový rozdíl (Subtract position.Z → **Absolute** → na druhou) [(7:49)](https://www.youtube.com/watch?v=5bbKHksth9Q&t=469s), horizontální vzdálenost (oběma bodům Set Attribute position.Z = 0, vektorová Distance → na druhou) [(8:35)](https://www.youtube.com/watch?v=5bbKHksth9Q&t=515s) [(9:21)](https://www.youtube.com/watch?v=5bbKHksth9Q&t=561s), podíl × 0,5 + 0,5 = **alpha** [(10:07)](https://www.youtube.com/watch?v=5bbKHksth9Q&t=607s). **Lerp** mezi pozicemi obou bodů s touto alphou vrátí midpoint vychýlený k nižšímu konci [(10:55)](https://www.youtube.com/watch?v=5bbKHksth9Q&t=655s). Spuštění dolů: Transform Points s offsetem Z = −min/max hang (Make Vector Attribute + multiply ×−1) a hlavně **absolute offset = true** — kliky na šikmém povrchu nesou rotaci povrchu a „dolů" by jinak mířilo šejdrem [(11:41)](https://www.youtube.com/watch?v=5bbKHksth9Q&t=701s) [(12:27)](https://www.youtube.com/watch?v=5bbKHksth9Q&t=747s). Nakonec **Merge Points se třemi piny v pořadí**: první bod, midpoint, druhý bod — pořadí pinů je pořadí splinu [(10:55)](https://www.youtube.com/watch?v=5bbKHksth9Q&t=655s).

**Ze tří bodů liána:** Create Spline → **Spline Sampler** (mode distance 100) s **unbounded + fit to curve** — fit dorovná rozestup, aby první a poslední bod seděly přesně na klicích, jinak poslední segment upadne [(14:00)](https://www.youtube.com/watch?v=5bbKHksth9Q&t=840s) [(14:47)](https://www.youtube.com/watch?v=5bbKHksth9Q&t=887s). Body pak dostanou offset ±20 **jen v Y a Z** — X vede podél splinu a rozbíjet ji nechceš — což dá lianě přirozené vlnění i kolísající tloušťku [(14:47)](https://www.youtube.com/watch?v=5bbKHksth9Q&t=887s) [(16:20)](https://www.youtube.com/watch?v=5bbKHksth9Q&t=980s). Mesh: Transform Points scale 0,1–0,5 → Create Spline → **Spawn Spline Mesh** (válec, forward axis Z, materiál jen napřímo — atributem přepsat nejde) [(15:33)](https://www.youtube.com/watch?v=5bbKHksth9Q&t=933s) [(17:06)](https://www.youtube.com/watch?v=5bbKHksth9Q&t=1026s).

**Dekorace ve třech hustotách:** z téže splinu tři Spline Samplery (distance 50 / 25 / 150, unbounded + fit to curve) s **max random offset normalized 0,8** — rozestupy přestanou být metronomické [(17:06)](https://www.youtube.com/watch?v=5bbKHksth9Q&t=1026s) [(17:53)](https://www.youtube.com/watch?v=5bbKHksth9Q&t=1073s). Každý sampler → Transform Points + Static Mesh Spawner: listnaté kusy s absolute rotation X −90 (visí dolů) a Z 360° dokola, scale 1–3; druhá sada Y ±30, scale 3–5; svislé šlahouny rotace 180 a scale 4–10 [(19:25)](https://www.youtube.com/watch?v=5bbKHksth9Q&t=1165s) [(20:57)](https://www.youtube.com/watch?v=5bbKHksth9Q&t=1257s) [(21:43)](https://www.youtube.com/watch?v=5bbKHksth9Q&t=1303s). Megascans LOD chyba z 5.7 (otočení o 90° při přepnutí) se tu léčí radikálněji: **number of LODs = 1** [(18:39)](https://www.youtube.com/watch?v=5bbKHksth9Q&t=1119s).

**Známá vada párování:** dvojice drží na indexech — smazání bodu **uprostřed** přepáruje všechno za ním a liány se překříží. Mazat od konce, nebo překreslit [(23:17)](https://www.youtube.com/watch?v=5bbKHksth9Q&t=1397s).

> **Pozn.:** Odzoomuj od lián: graf je obecný **nástroj na závěsné křivky** — dva kliky, spočtený průvěs, spline a spawn čehokoli podél. Lanový most, vlajky mezi domy, vánoční světla [(24:03)](https://www.youtube.com/watch?v=5bbKHksth9Q&t=1443s). A proč megascans místo nových procedurálních stromů? Autor to říká napřímo: systém je experimentální a vlastní assety do něj zatím nedostaneš [(22:30)](https://www.youtube.com/watch?v=5bbKHksth9Q&t=1350s) — viz [stejný limit u stromů](pcg-vegetace.md#proceduralni-stromy-v-enginu-57).
