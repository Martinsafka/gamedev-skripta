# Zápisky

Lekce z vlastních projektů — deníky vývoje a shrnutí pracovních debat, destilované do čitelné podoby. Všechno anonymizované: **identity nejsou pro myšlenku důležité.** Každý zápisek nese strukturu kontext → co se stalo → co si z toho beru → souvislosti; surovina zůstává mimo repozitář, publikuje se destilát.

Kde se s teorií a praxí rozcházejí, říkají to nahlas — reálná zkušenost má přednost před převzatou moudrostí.

**Filozofie a metoda**

- **[Selhávat citelně, selhávat čitelně](selhavat-citelne-a-citelne.md)** — maxima celých Zápisků: učí jen selhání, které bolelo a je zapsané; smyčka je motor, nadšení jen palivo; mantinely se zjišťují testem.
- **[Karta selhání: packaging visel na dvou shaderech](karta-selhani-packaging.md)** — tříhodinový zásek (IncrediBuild bez licence), jeho pitva — a šablona karty selhání, která z něj vznikla.
- **[Dokumentace jako audit](dokumentace-jako-audit.md)** — psaní reference k vlastní komponentě vytáhlo 14 nálezů dluhu: write-path se testuje ne-defaultními hodnotami, piš z artefaktu, ne ze záměru.

**Projekty a rozhodnutí**

- **[Derivační řetěz: od zadání k žánru](derivace-izby.md)** — jak vznikla IZBA: zadání sobě samému, mechaniky s alternativami a zamítnutími, narativ odvozený nakonec — a žánr jako one-way door s premisami.
- **[Lom, ne hřbitov](lom-ne-hrbitov.md)** — přerostlý první koncept jako předek portfolia: dvě hry a dva nástroje vytěžené z jednoho monolitu; rozdíl mezi lomem a dokumentem.
